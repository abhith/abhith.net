You may think why such a post in 2018. This is because I recently moved to another firm where I have to support some legacy apps.
Few of them are built using **Visual Studio 2010** and they have no plan to upgrade so far. After working so long in VS2017 with ReSharper and all, working with VS2010 is not so pleasant. But in order to improve the default experience VS2010 offering, following extensions helped me so far.

### Go To Definition

[![BuildVision](https://vsmarketplacebadge.apphb.com/version-short/NoahRichards.GoToDefinition.svg)](https://marketplace.visualstudio.com/items?itemName=NoahRichards.GoToDefinition)
[![BuildVision](https://vsmarketplacebadge.apphb.com/installs-short/NoahRichards.GoToDefinition.svg)](https://marketplace.visualstudio.com/items?itemName=NoahRichards.GoToDefinition)
[![BuildVision](https://vsmarketplacebadge.apphb.com/rating-short/NoahRichards.GoToDefinition.svg)](https://marketplace.visualstudio.com/items?itemName=NoahRichards.GoToDefinition)

[Go To Definition](https://marketplace.visualstudio.com/items?itemName=NoahRichards.GoToDefinition) make ctrl+click perform a "Go To Definition" on the identifier under the cursor. Also, when the ctrl key is held down, highlight identifiers that look like they have definitions to navigate to.

### DPack

[DPack](http://www.usysware.com/dpack/Default.aspx) is a FREE collection of Microsoft Visual Studio tools. DPack is designed to greatly increase developer's productivity, automate repetitive processes and expand upon some of Microsoft Visual Studio features.
